## Defer Resolution in Favour of Better Generation

Proposed direction: rather than building a resolution stage now (to handle conflicts between proposals), invest in better generation strategies first. If the generation strategy produces well-categorised, domain-specific proposals via specialist readers, the resolution problem largely dissolves.

Proposed next steps:
1. Restore the reference wiki from git history
2. Build the evaluation framework (completeness, invariants, structure, categorisation checks)
3. Run the specialist reader approach as an alternative generation strategy
4. Compare output against the reference
